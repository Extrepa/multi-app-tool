## File 13: PROP_STATES_LOGIC.md

```markdown
# Multi-State Asset Management

## Concept
Assets are not always static. A single "Prop ID" can contain multiple "States" (e.g., Idle, Active, Broken).

## Data Structure
- `assets[].states`: An array of objects containing `state_name` and `svg_data`.
- `instances[].current_state`: A pointer to which state is currently rendered on the Stage.

## Cursor Instructions
1. **Focus Window:** Add a "State Manager" tab. Allow users to add/remove states for the current asset.
2. **Switching:** When a user clicks a state (e.g., "Broken"), the SVG editor updates to show that specific path data.
3. **Stage Canvas:** Provide a dropdown in the Inspector to change the `current_state` of the selected instance.

```

```markdown
# Scene Maker: Layout Intelligence

## Snapping & Alignment
- **Grid Snapping:** Standard 8px/16px/32px grid.
- **Object Snapping:** Use bounding-box collision detection to snap the edges of Prop A to Prop B.
- **Smart Guides:** Display temporary "Pink Lines" (Figma style) when the centers or edges of two props align.

## Mass Manipulation
- **Box Selection:** Click and drag to select multiple props.
- **Group/Ungroup:** Allow users to "Group" props. A Group gets its own UUID but moves all children simultaneously.
- **Distribution Tools:** Buttons for "Distribute Horizontally" and "Align Bottom."

```

## File 15: VIBE_TEMPLATES.md

```markdown
# Global FX & Vibe Templates

## Concept
Separate the "Vibe" (Animation/FX) from the "Asset" (SVG/PNG) so they can be reused.

## Workflow
1. **Create Template:** In the FX Lab, save a set of parameters (e.g., "Neon Flicker") as a `.vibe` template.
2. **Apply to Tags:** Any asset in the library tagged with "Neon" automatically inherits this vibe.
3. **Override:** If an instance on the Stage needs to be different, allow a "Local Override" of the template values.

## Implementation
- Store `vibe_templates` in the `project.library`.
- The renderer should check: `Instance.vibe_override || Asset.vibe_template || Default_Vibe`.

```
