### 1. The "Tiny Runtime" Philosophy

The goal of the Bridge is to recreate the math you've built in the **Vibe Lab** using raw SVG and Vanilla JS.

* **Zero Dependencies:** The exported code doesn't need React or Zustand; it uses a native `requestAnimationFrame` loop.
* **SVG Injection:** The exporter generates a clean SVG string and a small "Brain" (the JS snippet) that handles the math for the specific oscillators (Pulse, Float, Shake) you applied to that asset.

### 2. Implementation: The Code Generator (`src/utils/bridgeExporter.ts`)

This utility takes a selected instance and generates a "Self-Animating" HTML component.

// Resource: Runtime JS Generation

  const { vibes } = instance;

  ${asset.pathData} </div>

<script>
  (function() {
    const el = document.getElementById('vibe-${instance.id}');
    const startTime = performance.now();
    
    function animate() {
      const t = (performance.now() - startTime) / 1000;
      let transform = "";
      
      // Inject the exact math from Step #111-#124

  `;
};

### 3. Step #202: The Headless Renderer

While the exporter generates the code, the **Headless Renderer** ensures that the "Stage" can be previewed exactly as it will appear in the final export. This is a "What You See Is What You Get" (WYSIWYG) bridge.

* **Isolated Styles:** It strips away the editor's surgical borders and grid lines.
* **Production CSS:** It optimizes the SVG for web performance, minifying paths and flattening groups where possible.
