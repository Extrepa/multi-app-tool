### 1. The Minification Logic

When we edit paths manually, we often end up with redundant coordinates or long floating-point numbers (e.g., `42.50000000001`). This step cleans the "dirty" SVG data before it hits the **Runtime JS Bridge**.

* **Coordinate Rounding:** All path coordinates are rounded to 2 decimal places, which significantly reduces file size without losing visual precision.
* **Command Compression:** Relative commands (e.g., `l` instead of `L`) are used where they save characters.
* **Redundant Point Removal:** If two nodes in a path are at the exact same coordinate, the optimizer merges them.
* **Zero-Decimals:** If a coordinate is a whole number (e.g., `50.00`), it is stripped down to just the integer (`50`).

---

### 2. Implementation: The Path Purifier (`src/utils/svgOptimizer.ts`)

This utility is called during the **Export** process (Step #201) to "Surgically Purge" the path data.

```typescript
// Resource: Path Optimization Utility
export const optimizePath = (pathData: string): string => {
  // 1. Split path into commands and coordinates
  const commands = pathData.match(/[a-zA-Z]|[-+]?[0-9]*\.?[0-9]+/g);
  
  if (!commands) return pathData;

  const optimized = commands.map(cmd => {
    const num = parseFloat(cmd);
    if (isNaN(num)) return cmd; // Keep command letters (M, L, C)
    
    // 2. Round to 2 decimal places and strip trailing zeros
    return parseFloat(num.toFixed(2)).toString();
  });

  // 3. Join back into a compact string without extra spaces
  return optimized.join(' ').replace(/\s([a-zA-Z])/g, '$1');
};

```
