### 1. The Cloud Persistence Layer

Until now, you've been using the **JSON Serializer** (Step #150) to save files locally. In Phase 11, we integrate a cloud-native backend (like Supabase or Firebase) to host these manifests, enabling cross-device access and public sharing.

* **Scene Snapshots:** When you hit "Share," the **Studio Store** state is compressed and sent to the database.
* **Unique Hash Generation:** The system generates a unique 8-character hash (e.g., `vibe.studio/errl-2025`) that serves as the pointer to your specific data.
* **Version Control:** Every save creates a new timestamped version, allowing you to track the evolution of your scene.

### 2. Implementation: The Sharing Bridge (`src/features/cloud/ShareButton.tsx`)

This component handles the asynchronous handshaking between your local workspace in Oregon and the global server.

```tsx
// Phase 11: Cloud Upload Logic
export const ShareProject = () => {
  const { scene, library } = useStudioStore();
  const [shareUrl, setShareUrl] = useState('');

  const handleShare = async () => {
    // 1. Package the 'Manifest'
    const manifest = { scene, library, timestamp: Date.now() };

    // 2. Upload to Cloud Storage
    const { data, error } = await supabase.from('projects').insert([
      { data: manifest }
    ]).select();

    if (data && !error) {
      // 3. Generate the Unique Pointer
      const shortCode = generateShortCode(data[0].id);
      setShareUrl(`https://vibe.studio/view/${shortCode}`);
    }
  };

  return (
    <div className="share-panel bg-[#13131f] p-4 border-l border-[#222]">
      <h3 className="text-[#00ff9d] text-[10px] font-bold mb-4 tracking-widest">DEPLOY_TO_CLOUD</h3>
      <button 
        onClick={handleShare} 
        className="w-full bg-[#b026ff] p-2 text-xs font-black hover:shadow-[0_0_15px_rgba(176,38,255,0.6)] transition-all"
      >
        GENERATE_LIVE_LINK
      </button>
      {shareUrl && (
        <input 
          readOnly 
          value={shareUrl} 
          className="mt-2 w-full bg-black border border-[#333] p-1 text-[10px] text-gray-400 font-mono" 
        />
      )}
    </div>
  );
};

```

### 3. The "Live Player" (Receiver View)

We build a dedicated `/view/:id` route that acts as a **Lightweight Headless Player**.

* **Minimal Payload:** It doesn't load the Forge, the Inspector, or the heavy UI toolbars. It only loads the **Global Ticker** and the **SVG Renderer**.
* **Performance:** It uses the **Production Snippet** logic (Phase 6) to ensure the scene runs smoothly on mobile devices, ensuring Errl looks great even when viewed on a phone in the middle of the woods.
* **Social Metadata:** Every link includes OpenGraph tags, so when you share Errl on a messaging app, a "Surgical Preview" of the character appears automatically.
