## 1. The "Nested Component" Scene Graph

In a professional engine, a scene isn't just a flat list of items; it’s a **Tree**.

* **The Workflow:** You create a "Wheel" (SVG). You put four Wheels on a "Car Body" (Component). You put the Car into a "City Street" (Scene).
* **The Power:** If you go back to the SVG Creator and add "Spikes" to the Wheel, the Car updates, and the City Street updates instantly.

### File 30: SCENE_GRAPH_HIERARCHY.md

```markdown
# Scene Graph & Parent-Child Logic

## Concept
Every object on the Stage is a 'Node'. Nodes can have children. Transformations (Move, Rotate, Scale) applied to a Parent automatically propagate to all Children.

## Implementation for Cursor
1. **The Tree UI:** Add a 'Layers' panel that supports nesting. Users can drag 'Prop A' onto 'Prop B' to make B the parent.
2. **Matrix Math:** When rendering, calculate the 'Global Position' by multiplying the Parent's transform matrix by the Child's local matrix.
3. **Selection:** Clicking a child selects the child. Double-clicking selects the entire Parent Group.

```

## 2. The "Vibe-Check" Event System

This is where "FX Lab" becomes "Logic Lab." You need a way to make the SVGs react to the player or the environment.

### File 31: EVENT_TRIGGER_SYSTEM.md

```markdown
# Logic & Interactivity Triggers

## Workflow
In the FX Lab, users assign 'Listeners' to their Components.

| Trigger Type | Action Result |
| :--- | :--- |
| **OnHover** | Swap SVG State to 'Glow' |
| **OnClick** | Emit 'Player_Jump' Signal |
| **OnProximity** | Increase 'Pulse' Vibe speed |

## Implementation
- Store these as an `events[]` array in the Component JSON.
- For Flash/Phaser exports, map these directly to the engine's Event Listeners (`addEventListener` or `.on()`).

```
